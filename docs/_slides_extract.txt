--- SLIDE 6 ---
Key findings:
1.Strong C2C demand in SEA, but no dedicated auction platform → Carousell: negotiation / fixed price · Shopee &
Lazada: B2C-focused
2.Global auction leaders (e.g. eBay) offer mature features but weak SEA focus and high seller fees → Opportunity for
local positioning and a lower commission model
3.Most platforms treat discovery (search/categories) separately from bidding → Differentiator: combine bidding with
behaviour-based recommendations (CF) using bids, views, and watchlist
4.Trust mechanisms are table stakes across peer marketplaces → Escrow-style payment flow, ratings, and admin
moderation
6
Research Summary
AuctionHub = SEA-oriented C2C marketplace that combines real-time auctions with behaviour-based recommendations,
not fixed-price negotiation alone.

Primary Objective
Build a SEA-focused C2C auction platform (AuctionHub) combining real-time bidding, collaborative-filtering
recommendations, and trust & moderation for buyers, sellers, and administrators.
In Scope
Web platform with Buyer, Seller, Admin roles (RBAC)
Timed auctions (ascending / Dutch / blind) with real-time bid updates (SSE/polling)
Concurrent-safe bidding and auto-bid (proxy bidding)
Search, filters, sort, watchlist, and personalised recommendations (CF + trending fallback)
Simulated escrow order flow (pay → ship → confirm → rate)
Ratings, Q&A, reporting, and admin moderation (users, listings, categories)
Seller analytics and platform revenue model (commission + featured listings)
Out of Scope
Real payment gateways (Stripe, PayNow)
Native mobile apps (iOS/Android)
Live video streaming auctions (Whatnot-style)
Cross-border logistics / multi-language / blockchain fraud detection

--- SLIDE 18 ---
Personalised Recommendation System
Explainable, non-ML user-based collaborative filtering
How it works
Signals
Bid on an auction → weight 3
Watchlist → weight 2
Browse (detail page view) → weight 1
Algorithm
Find similar buyers (same bids / watchlist / browse
patterns)
Score auctions they liked but you haven’t seen
Cosine similarity ranks peer strength
Trending fills gaps for new or sparse users (cold start)
Where users see it
Homepage: “Recommended for You” (logged in) ·
“Popular Right Now” (guest)
API: GET /api/recommendations
Future sprint: hybrid / ML-enhanced recommendations (content features, tuning, detail-page widgets); current build stays explainable CF for the prototype

recommendations, and post-auction orders.
Seller - Listing and sales: create and manage auctions,

ratings, and refund handling.
Admin - Platform operations: health dashboard, user and
listing moderation, category management, and analytics /
report export.

--- SLIDE 20 ---
Defence in depth for AuctionHub
• RBAC : Buyer, Seller, Admin enforced by filters & API
• Authentication : hashed passwords, Bearer tokens, TOTP 2FA
• Encryption : AES-256-GCM for sensitive fields at rest
• Safe data access : parameterised SQL, no IDOR from client IDs
• Browser hardening : security headers on all responses
• Privacy : masking & anonymised account deletion (PDPA)

Testing Methods

JUnit 5 + Mockito; API servlets and DAOs with mocked dependencies (~74 files, ~735 tests)

mvn test in FYP/ before merge and milestone submissions

4.Security-focused tests
RBAC matrices, SQL-injection strings, IDOR checks, encryption and masking assertions
5.Database & concurrency checks
Transaction rollback, account anonymisation; row-locked bidding design (SELECT … FOR UPDATE)
6.Black-box manual smoke (Render)
Login, search, SSE bidding, seller listing, admin moderation, GET /api/health
